— How It Integrates —

A layer between your AI and the code suggestion.

AlignIQ doesn't replace ambient AI documentation. It sits between the AI's clinical NLP engine and the code suggestion that reaches the coder — evaluating PSI-90 risk in real time, holding any suggestion that fails its guardrail conditions.

Before any PSI-90 code is suggested, AlignIQ checks 4 conditions per measure. ALL conditions TRUE → release. ANY false or unknown → hold + CDI query.

Released suggestions appear normally. Held suggestions are flagged with the reason and the auto-generated CDI query, escalating to the provider for clarification.
Deployment model: AlignIQ runs as a deterministic logic layer — no model retraining required, no provider workflow disruption, no impact on your AI's existing transcription or summary outputs. Only PSI-90 code suggestions pass through the guardrail.
